# DEMO_SCRIPT.md

# Takt — сценарий защиты на 5–7 минут

## 0. Подготовка перед показом

```bash
cd ~/Projects/takt-ai-control-center
docker compose up -d
docker compose ps
```

Открыть:

- Frontend: http://localhost:5173
- Backend API: http://localhost:8000/docs

Выбрать роль **Администратор**.

---

## 1. Проблема — 30–40 секунд

Компании внедряют LLM и AI-агентов в разные процессы, но обычно не видят в одном месте:

- сколько реально стоят AI-системы;
- какие модели и инструменты используют агенты;
- где возникают повторные вызовы и потери;
- какой бизнес-результат дают расходы;
- какие нарушения политик происходят;
- кто имеет доступ к данным и управлению.

Takt объединяет AI Observability, AI FinOps и Governance.

---

## 2. Реестр AI-систем — 60 секунд

Открыть **Продукты**.

Показать:

- продукт;
- связанного агента;
- deployment;
- окружение dev/prod;
- модель и тариф.

Нажать **Подключить AI-систему** и объяснить мастер:

1. создаёт AI-продукт;
2. регистрирует агента;
3. создаёт deployment;
4. сохраняет тариф модели;
5. выпускает отдельный telemetry API key;
6. формирует готовый SDK snippet.

Необязательно создавать новый объект во время защиты, если времени мало.

---

## 3. Наблюдаемость и Run Details — 90 секунд

Открыть **Запуски**.

Показать:

- статус запуска;
- токены;
- задержку;
- стоимость;
- окружение;
- время выполнения.

Открыть один запуск.

В Run Details показать:

- provider и модель;
- input/output/cached/reasoning tokens;
- стоимость каждого LLM-вызова;
- tool calls;
- retries;
- business outcome;
- human acceptance;
- нарушения политик;
- итоговую сверку стоимости.

Главная формула:

```text
Run cost = LLM cost + Tool cost + прочие зарегистрированные расходы
```

Важно: cached tokens входят в input tokens, reasoning tokens входят в output tokens и повторно не суммируются.

---

## 4. Экономика — 60 секунд

Открыть **Экономика**.

Показать:

- общие расходы;
- стоимость одного запуска;
- стоимость полезного результата;
- число токенов;
- waste;
- прогноз бюджета;
- самые дорогие продукты и агенты.

Объяснить:

```text
Cost per outcome = Total cost / Successful outcome quantity
```

```text
Waste = failed/cancelled runs
      + отклонённые outcomes
      + зарегистрированные retry costs
```

---

## 5. Управленческий отчёт — 60 секунд

Открыть **Отчёты**.

Показать:

- расходы по дням;
- потери;
- продукты и агенты;
- business value;
- time saved;
- net effect;
- ROI;
- автоматические управленческие выводы.

Формулы:

```text
Net effect = Estimated business value - Total cost
```

```text
ROI = (Estimated business value - Total cost) / Total cost
```

Нажать **Печать / PDF** и показать готовность отчёта для руководителя.

---

## 6. RBAC и Governance — 60 секунд

Переключиться на **External Auditor**.

Показать, что доступны только:

- Отчёты;
- Риски;
- Политики;
- Аудит.

Недоступны:

- экономика;
- продукты;
- запуски;
- бюджеты;
- интеграции;
- управление доступами.

Подчеркнуть: ограничения действуют не только во frontend, но и на backend API — прямой запрос возвращает `403`.

---

## 7. Завершение — 20 секунд

Takt отвечает на три вопроса:

1. **Что делают AI-системы?**
2. **Сколько это стоит?**
3. **Какой результат и риск получает бизнес?**

MVP уже показывает полный путь:

```text
Registry → Telemetry → Runs → Cost → Outcome → Governance → Report
```
